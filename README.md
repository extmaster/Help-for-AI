# Руководство по настройке ИИ-ассистентов

Это модульное и интерактивное руководство, созданное для эффективной настройки и использования современных ИИ-ассистентов, таких как OpenCode и Zed Editor. Проект охватывает ключевые концепции, лучшие практики и пошаговые инструкции для максимально продуктивной работы с искусственным интеллектом в вашей разработке.

**[Посмотреть руководство на GitHub Pages](https://extmaster.github.io/Help-for-AI)**


## Особенности проекта

*   **Модульная структура:** Контент разбит на независимые HTML-файлы (секции), что упрощает навигацию, редактирование и масштабирование.
*   **Динамическая загрузка (Pseudo-SPA):** Основная страница (`index.html`) динамически подгружает содержимое секций с помощью JavaScript, обеспечивая плавный пользовательский опыт без перезагрузки всей страницы.
*   **Автоматическая навигация:** Специальный скрипт `build.js` автоматически сканирует папку `sections/` и генерирует навигационное меню, избавляя вас от ручного обновления ссылок.
*   **Актуальный контент:** Подробно рассматриваются:
    *   **MCP (Model Context Protocol):** Открытый стандарт для интеграции ИИ с внешними системами.
    *   **AGENTS.md / Rules:** Создание пользовательских правил и инструкций для ИИ.
    *   **Skills:** Разработка и использование переиспользуемых навыков для агентов.
    *   **Настройка OpenCode и Zed Editor:** Практические рекомендации.
    *   **Работа с GitHub:** Приватные репозитории и GitHub Pages.
*   **Легкость развертывания:** Идеально подходит для хостинга на GitHub Pages.

## Как пользоваться

### 1. Клонирование репозитория

Для начала склонируйте репозиторий на ваш локальный компьютер:

```bash
git clone https://github.com/ВАШ_НИКНЕЙМ/ВАШ_РЕПОЗИТОРИЙ.git
cd ВАШ_РЕПОЗИТООРЙ
```

### 2. Первая сборка навигации

После клонирования **обязательно** запустите скрипт `build.js` из корневой папки проекта. Это создаст файл со списком всех секций и сгенерирует навигационное меню:

```bash
node build.js
```
*Вам потребуется установленный [Node.js](https://nodejs.org/), чтобы выполнить эту команду.*

### 3. Локальный просмотр (Опционально)

Для просмотра сайта на вашем компьютере (перед отправкой на GitHub), рекомендуется использовать простой локальный сервер, чтобы JavaScript работал корректно (из-за ограничений безопасности браузера при открытии файлов по протоколу `file://`).

**Рекомендуемый способ:** Установите расширение **"Live Server"** для VS Code. Затем просто откройте проект в VS Code, кликните правой кнопкой мыши по `index.html` и выберите "Open with Live Server" (или нажмите на кнопку `Go Live` в нижней строке редактора).

### 4. Публикация на GitHub Pages

1.  Убедитесь, что ваш репозиторий **публичный**.
2.  Перейдите в **Settings** вашего репозитория на GitHub, затем в раздел **Pages**.
3.  В секции "Build and deployment" выберите **"Deploy from a branch"**.
4.  Выберите ветку `main` (или `master`) и папку `/root`.
5.  Нажмите **Save**.
6.  Ваш сайт будет доступен по адресу `https://ВАШ_НИКНЕЙМ.github.io/ВАШ_РЕПОЗИТОРИЙ/` через несколько минут.

## Рабочий процесс: Добавление и обновление контента

1.  **Создайте или отредактируйте файл-секцию:** Добавьте новый HTML-файл в папку `sections/` (например, `09-новая-секция.html`) или отредактируйте существующий. Используйте числовой префикс для правильной сортировки в навигации.
2.  **Запустите `build.js`:** В терминале, находясь в корневой папке проекта, выполните:
    ```bash
    node build.js
    ```
    Это обновит навигацию на сайте, включив новые или измененные секции.
3.  **Сохраните и опубликуйте изменения в Git:**
    ```bash
    git add .
    git commit -m "Добавлена новая секция: Название Секции" # Или другое описание изменений
    git push
    ```
