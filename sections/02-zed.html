2. Настройка Zed Editor

Шаг 1: Создание Rules

Создай файл в корне проекта (любое из названий):

  • .rules (рекомендуется для Zed)
  • AGENTS.md (универсальный)
  • CLAUDE.md (совместимость с Claude)

Пример содержимого:

# Мой проект

## Структура
- `src/` - исходный код
- `tests/` - тесты

## Стандарты кода
- TypeScript strict mode
- Функциональный стиль
- Комментарии на русском

## Важно
- Для API используй axios
- Тесты пишем на vitest

Шаг 2: MCP серверы в Zed

Открой settings.json (Cmd+, или Ctrl+,):

{
"context_servers": {
"context7": {
  "command": "npx",
  "args": ["-y", "@context7/mcp-server"]
},
"remote-mcp-server": {
  "url": "https://mcp.example.com",
  "headers": { "Authorization": "Bearer TOKEN" }
}
}
}

Или через расширения Zed:

  1. Cmd+Shift+P → zed: extensions
  2. Найди нужный MCP сервер (Context7, GitHub, Puppeteer)
  3. Установи и настрой API ключи

Шаг 3: Проверка работы MCP

  1. Открой Agent Panel
  2. Меню (...) → Settings
  3. Смотри индикатор рядом с сервером:
  • Зелёный = работает
  • Другой цвет = проблема
