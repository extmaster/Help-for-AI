5. Работа с GitHub

Работа с приватными репозиториями

Приватные репозитории видны только вам и тем, кому вы предоставили доступ. Это идеальное место для хранения кода проектов, конфигураций и других данных, которые не должны быть общедоступными.

1. Создание нового приватного репозитория
  1. На любой странице GitHub нажмите "+" в правом верхнем углу и выберите "New repository".
  2. Введите имя репозитория.
  3. Выберите опцию "Private".
  4. Нажмите "Create repository".
2. Подключение локального проекта к репозиторию

Если у вас уже есть проект на компьютере, который вы хотите загрузить:

# Перейдите в папку вашего проекта
cd /path/to/your/project

# Инициализируйте Git, если это не было сделано
git init

# Добавьте все файлы для первой загрузки
git add .
git commit -m "Initial commit"

# Свяжите локальный проект с удаленным репозиторием на GitHub
git remote add origin https://github.com/USERNAME/REPOSITORY_NAME.git

# Загрузите код в репозиторий
git push -u origin main

Приватный репозиторий готов к работе.

Публикация сайтов с помощью GitHub Pages

GitHub Pages — это сервис для хостинга статичных сайтов (HTML, CSS, JS) прямо из вашего репозитория. Он бесплатен для публичных репозиториев.

1. Типы сайтов
  • Сайт пользователя/организации: Создается из репозитория с именем username.github.io. Доступен по адресу https://username.github.io.
  • Сайт проекта: Создается из любого другого репозитория. Доступен по адресу https://username.github.io/repository-name.
2. Процесс публикации
  1. Создайте публичный репозиторий на GitHub.
  2. Добавьте в него файлы вашего сайта. Как минимум, должен быть файл index.html в корне.
  3. Загрузите файлы в репозиторий (git push).
  4. В настройках репозитория (Settings) перейдите в раздел Pages.
  5. В секции "Build and deployment" выберите источник (Source) как "Deploy from a branch".
  6. Выберите ветку (обычно main или master) и папку (обычно /root).
  7. Нажмите Save.

Через несколько минут ваш сайт будет доступен по указанному адресу.

3. Использование своего домена

Вы можете подключить собственный домен (например, www.my-cool-project.com).

  1. В настройках DNS вашего домена создайте запись CNAME, указывающую на username.github.io.
  2. В настройках GitHub Pages (Settings → Pages) в поле Custom domain введите ваш домен и нажмите Save.
